आदेश दर्पण काठमाडौं जिल्ला अदालतको दैनिक आदेशहरू हेर्न, खोज्न र निर्यात गर्न बनाइएको वेब अनुप्रयोग हो। JSON फाइलहरूबाट मुद्दाका आदेशहरू लोड गरी कार्ड ढाँचामा प्रस्तुत गर्छ।

📂 फाइल लोड

HTML फाइलको तलतिर <script src="DOCS/2080.01.03.json"> ट्यागहरू थपेर फाइलहरू लोड गरिन्छ। समर्थित ढाँचाहरू:

  • .json — तयारी डेटा, सिधै लोड (AI नचाहिने)
  • .txt / .doc / .docx / .rtf / .pdf — Claude Haiku AI ले पार्स गर्छ (API चाहिन्छ)
📋 JSON ढाँचा

प्रत्येक JSON फाइलमा यो संरचना हुनु पर्छ:

{
  "label": "2080.01.03.json",
  "cases": [
    {
      "caseNo": "080-C1-0234",     // मुद्दा नं.
      "petitionNo": "",           // निवेदन नं. (वैकल्पिक)
      "type": "देवानी",          // मुद्दाको किसिम
      "subject": "",             // विषय (वैकल्पिक)
      "court": "काठमाडौं जिल्ला अदालत",
      "judge": "ईश्वर पराजुली",
      "date": "२०८०।०१।०३",
      "order": "सम्पूर्ण आदेश पाठ…", // पूरा आदेश
      "recognition": "देवानी, पूर्ण इजलास, वादी दावी पुग्ने"
    }
  ]
}
⚠ recognition: कमाले छुट्याइएका किवर्डहरू राख्नुहोस् (जस्तै: देवानी, पूर्ण इजलास, सअ, वादी दावी पुग्ने)। यिनै किवर्डमा दाँया खोजले काम गर्छ।

🔍 बाँया खोज (मुद्दा खोज)

मुद्दा नं., निवेदन नं., किसिम, विषय र किवर्डमा खोज्छ।

  • स — सामान्यीकरण (default ON): श→स, ी→ि, व→ब आदि एउटै मानिन्छ
  • फो — फोल्डरमा मात्र: फाइल नाममा मात्र खोज्छ
  • मिलेका मुद्दाहरू स्वतः खुल्छन् र हाइलाइट हुन्छन्
  • Enter / Shift+Enter — अर्को/अघिल्लो नेभिगेट
  • Esc — खोज खाली गर्नुहोस्
📄 आदेश कार्डहरू (दाँया प्यानल)

प्रत्येक आदेश कार्डमा:

  • 🏷 किवर्ड (recognition)
  • आदेश पाठ — 📋 सम्पूर्ण आदेश कपी, 📄 मुख्य अंश मात्र कपी, ⬇ स्रोत फाइल डाउनलोड
  • 📅 मिति र ⚖️ न्यायाधीशको नाम

🏷 किवर्ड अपरेटरहरू

किवर्ड मोडमा कमाले छुट्याइएका प्रत्येक टोकनमा खोज हुन्छ:

अपरेटरअर्थउदाहरण
|OR — कुनै एकपूर्ण|आंशिक
||AND — दुवैपूर्ण||वादी
|||NOT — पहिलो छ, दोस्रो छैनपूर्ण|||नपुग्ने
( )समूह(पूर्ण||अवण्डा)|||नपुग्ने
💡 उदाहरण: (पूर्ण||अवण्डा)|||नपुग्ने — किवर्डमा पूर्ण र अवण्डा दुवै छन् तर नपुग्ने छैन भन्ने आदेशहरू देखाउँछ।
🔣 Regex (नियमित अभिव्यक्ति) खोज

🔣 Regex खोज checkbox ON गर्नुहोस्। ढाँचाहरू:

ढाँचाउदाहरणअर्थ
/pattern/flags/पूर्ण.*?सदर/iपूर्णदेखि सदर सम्म (non-greedy), case-insensitive
raw patternवादी.+?पुग्नेवादी र पुग्ने बीच केही (non-greedy)
.*पूर्ण.*सदरgreedy — बीचमा जति पनि
.*?पूर्ण.*?सदरnon-greedy — सकेसम्म कम
⚠ ध्यान: Regex मोडमा किवर्ड अपरेटरहरू (|, ||, |||) काम गर्दैनन्। Regex syntax मिलेन भने count badge मा Regex त्रुटि देखिन्छ।

🏷 किवर्ड (recognition) ढाँचा

JSON को recognition फिल्डमा कमाले छुट्याइएका किवर्डहरू राख्नुहोस्:

"recognition": "देवानी, घरजग्गा अबण्डा ठहर भई वण्डा गरिएको, सअ, पूर्ण इजलास, वादी दावी पुग्ने-उल्टी- उल्टी भई सुरू सदर"

प्रत्येक कमाको बीचको भाग एक किवर्ड टोकन हो। खोजले प्रत्येक टोकनमा छुट्टाछुट्टै मिलाउँछ — अपरेटरहरू टोकन स्तरमा काम गर्छन्।

💡 सुझाव: छोटा, अर्थपूर्ण किवर्डहरू राख्नुहोस् — जस्तै मुद्दाको किसिम, इजलास, नतिजा (पुग्ने/नपुग्ने), विशेष आदेश आदि।
